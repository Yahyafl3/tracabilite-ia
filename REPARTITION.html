Répartition des tâches — Traçabilité IA

Projet en 3 parties : backend (Spring Boot) · frontend (Angular) · ml-service (Flask / scikit-learn). Équipe : A = Yahya · B = binôme.

Principe : découpage vertical. Chacun prend une fonctionnalité complète de bout en bout (son propre morceau de backend + ml-service + frontend). Les deux verticales sont indépendantes → chacun travaille à son rythme, sans attendre l'autre, tout en touchant les 3 dossiers.

Stack technique

  • Backend : Spring Boot (Java 21) + PostgreSQL + Spring Security / JWT — squelette à construire.
  • Frontend : Angular 21 (standalone, SSR) — landing + login faits, à connecter au backend.
  • ml-service : Flask + scikit-learn — fonctionnel (/predict, /train, /domains, /health), à enrichir.

Règles de travail

  • 1 tâche = 1 responsable : une seule personne code ET teste sa tâche avant de la partager.
  • Auto-test obligatoire avant de dire « fini » : test unitaire + test réel (Docker / navigateur / Postman), documenté.
  • Rétro d'erreurs après chaque tâche (modèle en bas).
  • Rotation : chacun touche backend + frontend + ml-service.
